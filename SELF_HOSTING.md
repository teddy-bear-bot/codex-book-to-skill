# Self Hosting

## 1. 目标

Book-to-Skill 的自部署方案应满足以下要求：

- 单机可跑通最小闭环
- 后续可平滑拆分为多服务
- 不绑定单一云厂商
- 不绑定单一模型服务商

## 2. 最小部署单元

建议最小可用部署包含：

- 1 个 Web/API 服务
- 1 个 Worker 服务
- 1 个关系型数据库
- 1 个对象存储

本地开发阶段可退化为：

- 单容器或单进程应用
- 本地文件存储
- 本地数据库

## 3. 推荐拓扑

### 3.1 开发环境

- Web/API: 本地运行
- Worker: 本地运行
- DB: Docker 容器
- Storage: 本地目录或 S3 兼容服务

### 3.2 生产环境

- Web/API: 独立部署
- Worker: 独立部署
- DB: 托管或自建关系型数据库
- Storage: S3 兼容对象存储
- Queue: 独立消息队列或任务系统

## 4. 必要配置

至少需要以下配置项：

- `APP_BASE_URL`
- `DATABASE_URL`
- `OBJECT_STORAGE_ENDPOINT`
- `OBJECT_STORAGE_BUCKET`
- `OBJECT_STORAGE_ACCESS_KEY`
- `OBJECT_STORAGE_SECRET_KEY`
- `QUEUE_URL`
- `MODEL_PROVIDER`
- `MODEL_API_KEY`

## 5. 文件与数据策略

- 上传文件不应写死在本地单一路径
- 导出文件应有生命周期管理
- 中间缓存应可清理
- 日志中不得打印原文全文和密钥

## 6. 模型供应商抽象

部署实现应支持模型后端替换：

- OpenAI 兼容接口
- 本地模型服务
- 其他第三方模型 API

建议通过统一 Provider 接口解耦：

- 文本抽取增强
- 方法论蒸馏
- Skill 生成
- Playground 调用

## 7. 运行模式建议

### 7.1 单机模式

适合：

- 本地开发
- 个人自部署
- 小规模测试

特点：

- 结构简单
- 运维成本低
- 可快速验证流程

### 7.2 分离式模式

适合：

- 长任务较多
- 文件较大
- 并发较高

特点：

- Web/API 与 Worker 分离
- 更易扩展
- 更方便观测和限流

## 8. 安全建议

- 所有密钥通过环境变量或密钥管理服务提供
- 上传文件设置访问权限
- 发布页与私有文件访问分离
- 增加任务级审计日志
- 公开实例默认关闭调试接口

## 9. 版权与合规建议

- 自部署不意味着可忽略版权边界
- 默认不公开展示全文
- 管理员应对公开实例设置引用长度限制
- 示例数据应使用可合法分发的文本
