# Web MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local Web MVP that lets users upload documents, preview a distilled Skill Spec, package it, and download a `.tar.gz` skill archive.

**Architecture:** Use a zero-dependency Node.js HTTP server with static HTML/CSS/JS. Add a focused document extraction layer for `.txt`, `.md`, `.docx`, `.doc`, and `.pdf`; text and Markdown read directly, DOC/DOCX use local conversion where possible, DOCX has an XML fallback, and PDF uses local best-effort extraction tools when available. The Web API reuses existing `src/distill`, `src/from-spec`, and `src/publish` modules rather than duplicating packaging logic.

**Tech Stack:** Node.js built-ins, POSIX/macOS local tools (`textutil`, `unzip`, `strings` fallback), `node:test`, browser HTML/CSS/JS
