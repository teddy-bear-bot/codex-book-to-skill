# From Spec Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `book-to-skill from-spec` to convert a local `skill-spec.json` file into a local skill package directory that works with the existing inspect, validate, and publish pipeline.

**Architecture:** Reuse the current package skeleton conventions, but derive files from the input spec rather than fixed placeholders. The library reads a local spec JSON file, validates the minimum fields needed for generation, maps spec fields into `skill.json`, `system.md`, `inputs.schema.json`, `README.md`, and `spec.json`, and returns any non-blocking warnings. CLI output stays concise and surfaces warnings after creation.

**Tech Stack:** Node.js built-ins, `node:test`
